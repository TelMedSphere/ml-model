<h2>ğŸ“ŒIntroduction</h2>

TelMedSphere is designed to make healthcare simple and accessible for both doctors and patients. It allows patients to connect with doctors through video calls, manage their health records, and make payments easily. For doctors, the app provides tools to schedule consultations, write prescriptions, and manage their time effectively.

<h2>ğŸ’¡Features</h2>

ğŸš¨ For Patients:<br>
 - Book Video Calls: Easily schedule video consultations with doctors.
 - Share Feedback: Rate and review the doctor after your consultation.
 - Manage Your Profile: Update and view your personal details.
 - View Past Records: Check previous orders and prescriptions in one place.
 - Easy Payments: Use the wallet feature powered by Stripe for secure payments.
 
ğŸš¨ For Doctors:<br>
 - Set Up Your Profile: Add information about yourself and your services.
 - Manage Availability: Set your working hours for consultations.
 - Join Video Calls: Connect with patients at the scheduled time.
 - Write Prescriptions: Share prescriptions directly with patients after the consultation.
 - Queue System: Organize appointments efficiently with a smart queue feature.

<h2>ğŸ³Docker Setup</h2>

Docker provides an easier way to set up and run TelMedSphere with all its dependencies.

### Prerequisites
- Docker and Docker Compose [installed](https://www.docker.com/products/docker-desktop/) on your system
- Environment variables ready for configuration

### Steps to Run with Docker

1. Clone the repository same as above:

2. Update Enviroment variables:
   - Update Environment variables mentioned in docker-compose based on each project's `.env` respectively


3. Start the application using Docker Compose:
```bash
docker-compose up --build -d
```

The application will be available at:
- Frontend: http://localhost:3000
- Backend: http://localhost:5000

### Stopping the Application
```bash
docker-compose down
```

### Remove the Container
```bash
docker-compose kill
```

# Environment Setup Guide

The project contains a `.env.example` file in both the frontend and backend directories. To properly set up your local environment, you'll need to replace the example values in `.env.example` with actual values. Here's a guide on how to fill out each environment variable.

## 1. **DBURL (MongoDB URL)**

This is the URL to your MongoDB database. If you're using MongoDB Atlas, follow these steps:

1. Go to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas) and sign up or log in.
2. Create a new cluster if you don't have one already.
3. Navigate to **Database Access** and create a new database user with the appropriate permissions (e.g., `readWrite`).
4. In the **Network Access** tab, allow connections from your IP address (or use 0.0.0.0/0 for all IPs for testing).
5. Go to **Clusters**, click on **Connect**, and select **Connect your application**.
6. Copy the connection string provided (it should look like `mongodb+srv://<username>:<password>@cluster0.mongodb.net/test?retryWrites=true&w=majority`).
7. Replace `<username>` and `<password>` with the credentials you created in step 3.
8. Set the `DBURL` variable in your `.env` file with the full MongoDB Atlas URL.

```env
DBURL=mongodb+srv://<username>:<password>@cluster0.mongodb.net/your-database?retryWrites=true&w=majority
```

For a local MongoDB setup, you can install MongoDB locally and use `mongodb://localhost:27017/your-database-name`.

---

## 2. **HOST_EMAIL (NodeMailer)**

This is the email address used to send emails via NodeMailer. For example, if you're using Gmail:

1. Go to [Google's App Passwords](https://myaccount.google.com/apppasswords).
2. Select **Mail** as the app and **Other** as the device.
3. Generate a new password and copy it.
4. Set `HOST_EMAIL` to your email address (e.g., `youremail@gmail.com`) and `PASSWORD` to the generated app password.

```env
HOST_EMAIL=youremail@gmail.com
PASSWORD=your_app_password
PORT=587
```

---

## 3. **STRIPE_SECRET_KEY**

To obtain your Stripe Secret Key:

1. Go to [Stripe Dashboard](https://dashboard.stripe.com/).
2. In the Developers section, find **API keys**.
3. Copy the **Secret Key** (start with `sk_test_` for test environments).
4. Set the `STRIPE_SECRET_KEY` in the `.env` file.

```env
STRIPE_SECRET_KEY=your_stripe_secret_key
```

---

## 4. **DOMAIN (Backend Server after Deployment)**

This is the domain where your backend will be hosted. If you're deploying to a platform like Heroku, Vercel, or any cloud service, use the public domain provided.

For example:

```env
DOMAIN=https://yourdomain.com
```

---

## 5. **SECRET (JWT Secret for Encryption)**

This is a secret key used for encoding JWT tokens. You can generate a random string using a tool like [Random.org](https://www.random.org/strings/) or by using Node.js:

```javascript
require('crypto').randomBytes(64).toString('hex');
```

Set this in your `.env` file:

```env
SECRET=your_secret_key
```

---

## 6. **WHATSAPP (Notification API Token)**

If you're using a WhatsApp API service like Twilio, follow these steps:

1. Sign up for [Twilio](https://www.twilio.com/).
2. Go to the **Console** and copy your **Account SID** and **Auth Token**.
3. Set the `WHATSAPP` variable in your `.env` file.

```env
WHATSAPP=your_whatsapp_api_token
```

---

## Final Steps

1. Once all the variables are set, save the `.env` file.
2. For frontend configuration, follow similar steps in the frontend `.env.example` file.
3. Run the project as per the instructions in the documentation (e.g., `npm start`, `node server.js`).

---

By following these steps, you will have your environment configured to run the application both locally and for deployment
